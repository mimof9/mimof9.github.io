git

参考

Git： 免费开源的分布式版本控制系统

概念

  • 工作区 workspace
  • 暂存区 index/stage
  • 本地仓库 local repository
  • 远程仓库 remote repository

1. 配置

  • git config --list 查看当前配置
  • git config -e [--global] 编辑项目(全局)配置文件
  • git config [--global] user.name <usernam> 配置用户名
  • git config [--global] user.email <email> 配置邮箱

2. 创建版本库

  • git init [folder] 本地新建
  • git clone <url> 远程克隆

3. 暂存区

  • git add [-p] <file> <dir> ... 将工作区中指定文件或者目录添加到暂存。-p添加每个变化前，都会要求确认，对于同一个文件的多出变化可以实现多次提交
  • git add . 将当前目录下的工作区内容全部添加到暂存区
  • git rm [--cached] [-f] [-r] <file> <dir> ... 取消跟踪暂存区指定文件，并删除工作区下对应文件。–cached保留工作区文件。-f强制删除。-r递归删除目录
  • git mv <file-oldname> <file-newname> 给工作区文件改名并同步到暂存区

4. 本地仓库

  • git commit [<file> <dir> ...] [-v] -m <message> 提交暂存区指定文件到本地仓库，如果不指明文件则提交全部内容。-v显示diff信息
  • git commit -a 提交工作区自上次commit之后的变化到本地仓库
  • git commit --amend -m <message> 覆盖上一次提交，如果代码没变化可以用来改写message

5. 分支

  • git branch [-r] [-a] 查看本地分支。-r查看远程分支。-a查看所有分支
  • git branch <branch> [commitid]创建本地分支，可以以指定提交为基础创建
  • git branch -d <branch>删除本地分支
  • git branch -dr <branch>删除远程分支
  • git push <romote> [--delete] <branch> –delete删除远程分支
  • git checkout <branch> 切换到指定分支
  • git merge <branch> 合并指定分支到当前分支
  • git cherry-pick <commitid> 选择一个commmit合并进当前分支

6. 远程仓库

  • git remote add <remote> <url> 增加一个远程仓库并命名，一般将远程仓库命名为origin
  • git remote -v 显示所有远程仓库
  • git remote show <remote> 显示指定远程仓库信息
  • git push [-u] <remote> <branch> [--all] [--force] 将指定的本地分支推送到指定的远程仓库的对应分支。-u表明指定默认远程主机。省略分支则推送当前分支。–all推送所有分支。–force无视冲突，强行推送，不要使用
  • git pull <remote> [branch] 拉取指定远程仓库分支的内容，自动与对应本地分支合并
  • git fetch <remote> [branch] 取指定远程仓库分支的内容，不自动合并
